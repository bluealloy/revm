# TIP-1016 State Gas — Test Plan

TIP-1016 adds dual-limit gas accounting: **execution gas** (CPU work) is tracked
separately from **state gas** (storage creation, account creation, code deposit).

All tests use these state gas costs:

| GasId | Cost |
|---|---|
| `sstore_set_state_gas` | 20,000 |
| `new_account_state_gas` | 25,000 |
| `code_deposit_state_gas` | 200 / byte |
| `create_state_gas` | 32,000 |

## Category 7: Reservoir Refill

**Mechanism**: When execution result is HALT or REVERT (not OK), `handler_reservoir_refill(reservoir, state_gas_spent)` refills the reservoir. Formula: `new_reservoir = reservoir + max(0, state_gas_spent - reservoir)`. This accounts for state gas that exceeded the reservoir and had to be drawn from regular gas.

| 7.1 | `test_tip1016_reservoir_refill_revert_state_gas_less` | SSTORE(0,1); REVERT. State gas entirely from reservoir. | state_gas=20,000 < reservoir=30,000 | REVERT, remaining reimbursed, gas_used ~63,106 | reservoir unchanged (20k all from reservoir) |
