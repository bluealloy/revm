# TIP-1016 State Gas — Test Plan

TIP-1016 adds dual-limit gas accounting: **execution gas** (CPU work) is tracked
separately from **state gas** (storage creation, account creation, code deposit).

All tests use these state gas costs:

| GasId | Cost |
|---|---|
| `sstore_set_state_gas` | 200,000 |
| `new_account_state_gas` | 250,000 |
| `code_deposit_state_gas` | 1,000 / byte |
| `create_state_gas` | 330,000 |

## Category 7: Reservoir Refill

**Mechanism**: When execution result is HALT or REVERT (not OK), `handler_reservoir_refill(reservoir, state_gas_spent)` refills the reservoir. Formula: `new_reservoir = reservoir + max(0, state_gas_spent - reservoir)`. This accounts for state gas that exceeded the reservoir and had to be drawn from regular gas.

| # | Test Name | Scenario | state_gas_spent vs reservoir | Expected Result | Expected Refill |
|---|-----------|----------|------------------------------|------------------|-----------------|
| 7.1 | `test_tip1016_reservoir_refill_revert_state_gas_less` | SSTORE(0,1); REVERT. cap=MAX so reservoir=0. State gas (200k) deducted from regular gas. REVERT reimburses remaining. | state_gas=200,000, reservoir=0 | REVERT, remaining reimbursed | delta == STATE_GAS_SSTORE_SET, gas_used < gas_limit |
| 7.2 | `test_tip1016_reservoir_refill_revert_state_gas_more` | 2×SSTORE; REVERT. cap=MAX so reservoir=0. State gas (400k) deducted from regular gas. REVERT reimburses remaining. | state_gas=400,000, reservoir=0 | REVERT, remaining reimbursed | delta == 2×STATE_GAS_SSTORE_SET, gas_used < gas_limit |
| 7.3 | `test_tip1016_reservoir_refill_halt_state_gas_less` | SSTORE(0,1) with tight gas_limit=25,000 and cap=MAX (reservoir=0). OOG on regular gas during SSTORE execution. | gas_limit=25k, reservoir=0 | HALT(OOG), gas_used = gas_limit | remaining=0, spent=gas_limit |
| 7.4 | `test_tip1016_reservoir_refill_halt_state_gas_more` | 3×SSTORE with cap=30,000, gas_limit=100,000. Creates reservoir (regular_budget=9k, reservoir=70k).
